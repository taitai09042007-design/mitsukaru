## 1. Web の基本

### クライアント / サーバー
ネットワークのサービスを「頼む側」がクライアント、「提供する側」がサーバー。
レストランの客(クライアント)と店員(サーバー)の関係。
*みつかる君では*: ブラウザがクライアント、Render上のNode.jsがサーバー。

### フロントエンド / バックエンド
フロントエンド=利用者の目に見える部分(HTML/CSS/JS、ブラウザで動く)。
バックエンド=裏側の処理(データ保存、計算、サーバーで動く)。
*みつかる君では*: `public/` フォルダがフロント、`server.js` と `lib/` がバック。

### HTML (HyperText Markup Language)
ページの**構造**(見出し、ボタン、画像…)を書く言語。タグ `<h1>` などで部品を表す。

### CSS (Cascading Style Sheets)
ページの**見た目**(色、配置、大きさ)を指定する言語。
*みつかる君では*: `public/css/style.css` 1ファイルに全画面分のデザインがある。

### JavaScript (JS)
ページに**動き**をつけるプログラミング言語。ボタンが押されたらAPIを呼ぶ、結果を画面に出す、など。
Node.jsの登場でサーバー側にも使えるようになった。

### URL (Uniform Resource Locator)
インターネット上の資源の住所。`https://例.com/api/found?q=水筒` は
「プロトコル + ドメイン名 + パス + クエリパラメータ」に分解できる。

### ドメイン名 / IPアドレス
IPアドレスはコンピュータの数字の住所(例: 216.24.57.4)。
ドメイン名は人間用の別名(例: onrender.com)。変換するのがDNS。

### DNS (Domain Name System)
ドメイン名→IPアドレスの変換を行う、インターネットの電話帳。

### HTTP / HTTPS
ブラウザとサーバーの会話ルール(プロトコル)。リクエストとレスポンスを繰り返す。
HTTPSはHTTPを**TLSで暗号化**したもの。今はHTTPSが標準。

### リクエスト / レスポンス
頼むメッセージがリクエスト(GET /api/stats)、返事がレスポンス(200 OK + JSON)。

### ステータスコード
レスポンスの結果を表す3桁の数字。200=成功、400番台=頼んだ側のミス、500番台=サーバーのミス。

### TCP / TLS
TCP=データを正確に順番通り届ける土台の仕組み。
TLS=通信を暗号化する仕組み(`https://` の正体)。盗聴・改ざんを防ぐ。

## 2. API と データ

### API (Application Programming Interface)
プログラム同士の「窓口」。みつかる君のフロントは `/api/found` などの窓口を通じて
サーバーとデータをやりとりする。人間用の画面ではなく**プログラム用の入口**。

### REST / RESTful API
APIの設計スタイル。「URLで対象(リソース)を表し、HTTPメソッドで操作を表す」。
例: `GET /api/found/3` = 「拾得物3番を **取得**」、`POST /api/found` = 「拾得物を **作成**」。

### JSON (JavaScript Object Notation)
データを `{"id": 1, "color": "blue"}` のような形式で表す記法。
人間にも読めて、ほぼ全ての言語で扱える、API通信の標準フォーマット。

### クエリパラメータ
URLの `?` 以降に付ける追加情報。`/api/found?q=水筒&status=stored` の `q` と `status`。

### HTTPヘッダ
リクエスト/レスポンスに付く「メタ情報」。
*みつかる君では*: PINを `x-staff-pin` というヘッダで送る。セキュリティヘッダもここ。

### multipart/form-data
画像などのファイルをHTTPで送るときの形式。テキストとファイルを「区切り」で分けて一緒に送れる。
*みつかる君では*: 拾得物登録(写真+説明文)で使用。

### base64
バイナリデータ(画像など)を64種類の文字だけで表現する変換方式。
データは約1.33倍に膨らむが、テキストしか入れられない場所(JSONやDB)に画像を入れられる。
*みつかる君では*: Firebaseに画像を保存するときに使用。

### データベース (DB)
データを整理して保存し、検索できるようにする専用ソフト。
**リレーショナル型**(表形式、SQL)と **NoSQL型**(柔軟な形式)に大別される。
*みつかる君では*: NoSQLのFirebase Realtime Databaseを使用。

### NoSQL
表(テーブル)形式にこだわらないデータベースの総称。FirebaseはJSONツリー型。

### キャッシュ
一度取得したデータを手元に置いておき、2回目以降を高速化する仕組み。
*みつかる君では*: Firebaseから読んだ画像をメモリにキャッシュ。ブラウザにも
`Cache-Control` ヘッダで「1年間使い回してOK」と指示している。

### デバウンス (debounce)
連続して発生するイベントを「少し待ってまとめて1回」処理するテクニック。
*みつかる君では*: データ変更のたびではなく、1秒待ってからFirebaseに書き込む。

## 3. サーバーサイド

### Node.js
JavaScriptをサーバーで動かす実行環境。非同期I/Oが得意で、多数の同時接続に強い。

### npm (Node Package Manager)
Node.jsのライブラリ(パッケージ)を管理するツール。`npm install` で
`package.json` に書かれた依存ライブラリを一括インストールできる。

### Express
Node.jsの定番Webフレームワーク。ルーティング(URL→処理の対応付け)とミドルウェアが中心機能。

### ルーティング
「このURL+メソッドに来たら、この関数を実行する」という対応表。
*みつかる君では*: `app.get('/api/stats', ...)` のような行がすべてルーティング。

### ミドルウェア
リクエストが本処理に届く前に通る「関所」関数。検証・認証・ログなど横断的な処理を担当。
*みつかる君では*: セキュリティヘッダ付与、JSON解析、レート制限、画像受信(multer)。

### multer
Expressで `multipart/form-data`(ファイルアップロード)を受け取るためのミドルウェア。
*みつかる君では*: メモリ上に受信(`memoryStorage`)→画像ストアへ保存、8MB制限とMIME検証付き。

### 非同期処理 / async・await
時間のかかる処理(通信、ファイル読み書き)の完了を**待っている間に他の仕事を進める**仕組み。
`async` 関数の中で `await` と書くと「終わるまで待つ(でも他のリクエストは止めない)」。
*みつかる君では*: Firebaseへの保存・読込はすべて非同期。

### 環境変数
プログラムの**外側**から渡す設定値。コードを書き換えずに動作を変えられ、
秘密情報(PIN、鍵)をコードに直書きしなくて済む。
*みつかる君では*: `STAFF_PIN`、`FIREBASE_DATABASE_URL` など。`.env` ファイルから読み込む。

### ポート番号
1台のコンピュータの中の「窓口番号」。Webは80(HTTP)/443(HTTPS)が標準。
*みつかる君では*: 開発時は3100番で待ち受け。

### WebSocket / Socket.IO
WebSocket=接続をつなぎっぱなしにして双方向通信する技術。
Socket.IO=WebSocketを使いやすくしたライブラリ(部屋分け、自動再接続つき)。
*みつかる君では*: マッチ通知(`match-found`)と一覧の自動更新に使用。

### ルーム (room)
Socket.IOの機能。接続中のクライアントをグループ分けし、特定グループだけに送信できる。
*みつかる君では*: 探し物ごとに `lost-1` のような部屋を作り、本人だけに通知。

## 4. AI・機械学習

### 機械学習 (Machine Learning)
ルールを人間が書く代わりに、**大量のデータから法則をコンピュータに学ばせる**手法。

### 学習済みモデル
すでに訓練が終わった機械学習の「頭脳」ファイル。自分で訓練しなくても、
ダウンロードしてすぐ予測に使える。

### 物体検出 (Object Detection)
画像の中の「何が・どこに」あるかを当てるタスク。分類(何が写っているか)より一歩進んで
**位置(枠)** も当てる。

### COCO-SSD
COCO(Common Objects in Context)データセットで訓練されたSSD方式の物体検出モデル。
80種類の日用品(bottle, umbrella, backpack…)を検出できる。
*みつかる君では*: 落とし物の写真からカテゴリを自動判定。

### TensorFlow.js
Googleの機械学習ライブラリTensorFlowの**ブラウザ版**。JavaScriptだけで
モデルの実行(推論)ができる。

### 推論 (Inference)
学習済みモデルに新しいデータを入れて予測させること。「学習」と対になる言葉。
*みつかる君では*: 写真→「bottle 92%」を出す処理が推論。

### エッジAI
AIの推論をクラウドではなく**端末側(エッジ=末端)**で行う方式。
通信不要・低遅延・プライバシー保護・コストゼロが利点。
*みつかる君では*: ブラウザ内でAIが完結するのが最大の特徴。

### Canvas API
ブラウザで画像や図形をピクセル単位で操作できるJavaScriptのAPI。
*みつかる君では*: 写真のピクセルの色を集計して「主要な色」を推定。

### 正規化 (テキスト処理)
表記のゆらぎ(全角/半角、カタカナ/ひらがな)を揃えてから比較する前処理。
*みつかる君では*: 「スイトウ」「すいとう」を同じものとして扱うために使用。

### 同義語辞書 / トークン化
同義語辞書=同じ意味の言葉のグループ表(水筒=ボトル=タンブラー)。
トークン化=文章を単語(トークン)に分割すること。
*みつかる君では*: `lib/synonyms.js` が辞書、検索文の分解がトークン化。

## 5. セキュリティ

### XSS (クロスサイトスクリプティング)
入力欄に `<script>悪意のコード</script>` を仕込み、他人のブラウザで実行させる攻撃。
**対策=エスケープ**: `<` を `&lt;` に変換して「ただの文字」として表示する。
*みつかる君では*: 表示する全データを `esc()` 関数でエスケープ。

### レート制限 (Rate Limiting)
同じ相手からのリクエスト回数を制限する仕組み。スパム・連打・総当たり攻撃を防ぐ。
*みつかる君では*: IPアドレスごとに「登録15回/分」「履歴閲覧10回/分」など。

### 総当たり攻撃 (Brute Force)
パスワードやPINを片っ端から試す攻撃。
*みつかる君では*: PIN付きAPIに厳しめのレート制限をかけて対策。

### バリデーション (入力値検証)
受け取ったデータが正しい形式・範囲かをサーバーで検査すること。
「クライアントは信用しない」が鉄則(ブラウザ側のチェックは突破できるため)。
*みつかる君では*: 文字数制限、カテゴリ/色のホワイトリスト照合。

### ホワイトリスト方式
「許可するものを列挙し、それ以外は全部拒否」する方式。ブラックリスト(禁止リスト)より安全。
*みつかる君では*: カテゴリは定義済みIDのみ、画像はJPEG/PNG/WebPのみ受付。

### 認証 (Authentication)
「あなたは誰?」を確認すること。パスワード、PIN、生体認証など。
*みつかる君では*: STAFF_PINによる簡易認証。

### セキュリティヘッダ
ブラウザに安全上の指示を伝えるHTTPヘッダ。
- `X-Frame-Options: DENY` — 他サイトのiframe内に埋め込ませない(クリックジャッキング対策)
- `X-Content-Type-Options: nosniff` — ファイル種別の勝手な推測を禁止
- `Referrer-Policy: no-referrer` — 遷移元URLを外部に漏らさない

### 最小権限 / 情報の最小公開
「必要な人に、必要な情報だけ」を渡す設計原則。
*みつかる君では*: 名前(届けた人・受け取った人)は一般向けAPIから除外し、PIN保護のAPIのみが返す。

### サービスアカウント鍵
プログラムがクラウド(Firebase等)に接続するための「機械用パスワード」。JSONファイル形式。
**絶対にGitHubに上げてはいけない**(`.gitignore` で除外+本番は環境変数で渡す)。

## 6. 開発・運用

### Git
コードの変更履歴を記録する**バージョン管理システム**。
「いつ・誰が・何を変えたか」が全部残り、いつでも過去に戻れる。

### コミット (commit)
変更履歴の1単位。「セーブポイント」のようなもの。メッセージを添えて記録する。

### リポジトリ (repository)
Gitで管理されるプロジェクトの保管庫。手元(ローカル)とGitHub上(リモート)の2つがある。

### プッシュ (push)
ローカルのコミットをリモート(GitHub)へ送ること。⇔ プル(pull)は逆方向。

### GitHub
Gitリポジトリをインターネット上でホスティングするサービス。共有・公開・自動連携の起点になる。

### デプロイ (deploy)
作ったプログラムを本番サーバーに配置して、実際に使える状態にすること。

### PaaS (Platform as a Service)
「コードを渡せば動かしてくれる」クラウドサービス。サーバー機の管理が不要。
*みつかる君では*: Renderを使用。GitHub連携で自動デプロイ。

### CI/CD (継続的インテグレーション/継続的デプロイ)
push のたびに自動でテスト(CI)・自動で公開(CD)する開発手法。
*みつかる君では*: push→Renderが自動ビルド&公開、がCDにあたる。

### Infrastructure as Code (IaC)
サーバーの構成を設定ファイル(コード)で管理する考え方。再現可能で間違いが減る。
*みつかる君では*: `render.yaml`(Blueprint)がこれ。

### エフェメラル(揮発性)ストレージ
再起動すると消えてしまう一時的な保存領域。クラウドの無料プランに多い。
*みつかる君では*: Renderがこれなので、データは外部のFirebaseに保存して対策。

### ローカル / 本番(プロダクション)
ローカル=自分のPC上の開発環境。本番=実際の利用者が使う公開環境。
設定(環境変数)を変えるだけで同じコードが両方で動くのが理想。

### 自動テスト
プログラムの動作確認をコードで自動化したもの。修正のたびに人手で全部試さなくて済み、
「直したつもりが別の場所を壊した」(デグレ)を検出できる。
*みつかる君では*: `test-flow.js` が登録→検索→マッチ→返却→履歴の16項目を自動検証。

### フォールバック
本来の手段が使えないときの「代替手段」。
*みつかる君では*: Firebase設定がなければ自動でローカルJSON保存に切り替わる。
