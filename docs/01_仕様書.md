# 📘 第1巻: みつかる君 システム仕様書

このドキュメントは「みつかる君」が **何をするシステムで、どう動くのか** を正確に定義した仕様書です。
コンテストの審査員への説明や、機能を追加するときの「正解の定義」として使えます。

---

## 1. システム概要

| 項目 | 内容 |
|---|---|
| システム名 | 落とし物マッチングAI「みつかる君」 |
| 目的 | 学校の落とし物が持ち主に届かない問題を解決する |
| 利用者 | ①拾った人 ②落とした人 ③保管担当(先生・委員) |
| 特徴 | 写真を撮るだけのAI自動タグ付け+ことばで検索+リアルタイム通知 |

### 解決する問題

従来の落とし物管理には3つの壁があります。

1. **登録の壁** — 拾っても届けるのが面倒(紙に書く、係に渡す…)
2. **発見の壁** — 落とし物棚を全部見に行かないと見つからない
3. **時間の壁** — 自分が探しに行ったときには、まだ届いていないことがある

みつかる君はこれを「**写真1枚で登録**」「**ことばで検索**」「**先に登録しておけば届いた瞬間に通知**」で解決します。

## 2. 画面一覧(フロントエンド)

| # | 画面 | URL | 役割 |
|---|---|---|---|
| 1 | さがす | `/` (index.html) | 自由文で落とし物を検索。新着もリアルタイム表示 |
| 2 | とどける | `/found.html` | 写真を撮る→AIが自動タグ付け→拾得物を登録 |

| 5 | 保管担当 | `/staff.html` | **PIN保護**。返却履歴(いつ・何を・誰に返したか)を確認 |

## 3. 機能仕様

### 3.1 拾得物の登録(とどける)

1. 利用者が写真を撮影 or 選択する
2. **ブラウザ内で** TensorFlow.js (COCO-SSDモデル) が物体検出を実行
   - 検出された物体名(英語: bottle, umbrella など)を日本語カテゴリに変換
   - Canvas で画像のピクセルを解析し、**主要な色** を推定
3. AIの推定結果がフォームに自動入力される(人間が修正可能)
4. 登録すると、サーバーが **待機中の探し物すべてとマッチング** を行い、
   スコアが閾値(50点)以上の人へ **Socket.IO でリアルタイム通知** を送る

> ポイント: AI処理は **すべて利用者のブラウザの中** で動く(エッジAI)。
> サーバーに画像認識の負荷がかからず、外部AI APIの料金もゼロ。

### 3.2 検索(さがす)

- 「青い水筒」のような **自由文** で検索できる
- サーバーがクエリを解析: カテゴリ(水筒)+色(青)+キーワード に分解
- 各拾得物にスコアをつけ、高い順に表示(スコアの内訳=マッチ理由も表示)

### 3.3 探し物の事前登録(通知をまつ)

- 「黒い折りたたみ傘」のように登録すると、まず **今ある在庫と即時マッチング**
- 見つからなければ「待機状態」になり、ブラウザは Socket.IO で待機部屋に入る
- 合う落とし物が届いた **瞬間** に `match-found` イベントで通知が届く
- 探し物リクエストは **30日で自動的に期限切れ**(古いデータが溜まらない)

### 3.4 返却の記録

- 拾得物カードの「返却する」ボタンで、誰に返したかを記録
- `STAFF_PIN` が設定されていれば **PINを知っている保管担当だけ** が記録できる

### 3.5 保管担当ページ(返却履歴)

- PIN認証後、返却済みアイテムの一覧を新しい順に表示
- 表示内容: 返却日時 / 品物(写真・カテゴリ・色) / **受け取った人** / 届けた人 / 拾った場所 / 保管場所
- 受け取った人・届けた人の名前は **このページ(API)でしか見られない**(プライバシー保護)

## 4. マッチングアルゴリズム仕様

検索も通知も、同じ **加点方式のスコアリング** で動きます。

| 評価項目 | 配点 | 説明 |
|---|---|---|
| カテゴリ一致 | **+50** | 「水筒」と水筒カテゴリの拾得物が一致 |
| 色一致 | **+25** | 「青」と拾得物の色が一致 |
| キーワード部分一致 | **+10/語(最大30)** | 説明文・タグ・場所に検索語が含まれる |

- **マッチ成立の閾値: 50点以上**(つまり最低でもカテゴリ一致が必要)
- 満点は 105点(50+25+30)

### 同義語辞書(synonyms.js)

日本語のゆらぎを吸収するための辞書を内蔵しています。

- カテゴリ: 「水筒/すいとう/ボトル/タンブラー」→ すべて `bottle` カテゴリ
- 色: 「青/あお/ブルー/ネイビー/水色」→ すべて `blue`
- 正規化: 全角/半角、カタカナ/ひらがなの違いを吸収してから比較

### AIラベルの変換(COCO → カテゴリ)

COCO-SSDが返す80種類の英語ラベル(`bottle`, `umbrella`, `backpack`…)を、
学校の落とし物カテゴリ(水筒、傘、カバン…)へ対応表で変換します。

## 5. API仕様(バックエンド)

ベースURL: サーバーのルート。すべて JSON でやりとりします。

### 5.1 メタ情報

```
GET /api/meta
```

フロントが使う設定情報を返す。

| フィールド | 説明 |
|---|---|
| `categories` | カテゴリ一覧(id, label, emoji, synonyms) |
| `colors` | 色一覧 |
| `cocoToCategory` | AIラベル→カテゴリの変換表 |
| `storagePlaces` | 保管場所の選択肢 |
| `matchThreshold` | マッチ閾値(50) |
| `staffPinRequired` | PINが設定されているか(true/false) |

### 5.2 拾得物

```
POST /api/found            … 拾得物の登録(レート制限: 15回/分)
GET  /api/found?q=&status= … 検索・一覧(q=自由文, status=stored|returned)
GET  /api/found/:id        … 1件取得
POST /api/found/:id/return … 返却記録(レート制限: 30回/分, PIN必須※)
```

**POST /api/found** は `multipart/form-data` 形式(画像を含むため)。

| パラメータ | 必須 | 説明 |
|---|---|---|
| `category` | ✅ | カテゴリID(不正な値は400エラー) |
| `color` | - | 色ID |
| `tags` | - | AIタグのJSON配列(最大10個、各30文字) |
| `description` | - | 説明(最大200文字) |
| `locationFound` | - | 拾った場所(最大100文字) |
| `storagePlace` | - | 保管場所 |
| `reporterName` | - | 届けた人(**一般には非公開**) |
| `image` | - | 画像(JPEG/PNG/WebP、8MBまで) |

レスポンス: `{ item: {...}, notified: 通知した人数 }`

**POST /api/found/:id/return** のボディ:

| パラメータ | 説明 |
|---|---|
| `returnedTo` | 受け取った人の名前(**一般には非公開**) |
| `staffPin` | ※STAFF_PIN設定時のみ必須。違うと401エラー |

### 5.3 探し物リクエスト

```
POST /api/lost              … 探し物の登録(レート制限: 15回/分)
GET  /api/lost/:id/matches  … マッチ結果の再取得
POST /api/lost/:id/resolve  … 「見つかった」として解決済みにする
```

**POST /api/lost** のボディ:

| パラメータ | 必須 | 説明 |
|---|---|---|
| `keywords` | ✅ | 探し物の説明文(最大100文字) |
| `category` | - | 省略時はキーワードから自動判定 |
| `color` | - | 省略時はキーワードから自動判定 |
| `contactName` | - | 名前(**APIレスポンスに含めない**) |

レスポンス: `{ request: {...}, matches: [{item, score, reasons}, ...] }`

### 5.4 保管担当

```
GET /api/staff/history     … 返却履歴(レート制限: 10回/分)
```

- 認証: HTTPヘッダ `x-staff-pin: <PIN>`(STAFF_PIN設定時)
- レスポンス: `{ items: [...返却済み一覧(returnedTo, reporterName入り)], storedCount: 保管中の数 }`
- レート制限が厳しめなのは **PINの総当たり攻撃対策**

### 5.5 統計・画像

```
GET /api/stats             … 統計(合計・返却率・カテゴリ別・直近7日)
GET /uploads/:filename     … 画像の配信(ローカル/Firebaseを自動切替)
```

### 5.6 Socket.IO イベント

| イベント | 方向 | 説明 |
|---|---|---|
| `watch-lost` | クライアント→サーバー | 探し物IDの「待機部屋」に入る |
| `match-found` | サーバー→クライアント | 待機部屋へ、マッチした拾得物を通知 |
| `found-registered` | サーバー→全員 | 新しい拾得物が登録された(一覧の自動更新用) |
| `item-returned` | サーバー→全員 | 返却された(一覧の自動更新用) |

## 6. データ構造

### 拾得物 (foundItem)

```json
{
  "id": 1,
  "category": "bottle",
  "color": "blue",
  "tags": ["bottle", "水筒"],
  "description": "青いステンレスの水筒",
  "locationFound": "体育館",
  "storagePlace": "職員室前の落とし物棚",
  "reporterName": "1-2 山田",
  "imagePath": "/uploads/found_1718...jpg",
  "status": "stored",
  "createdAt": "2026-06-11T01:00:00.000Z",
  "returnedAt": null,
  "returnedTo": null
}
```

- `status` は `stored`(保管中)→ `returned`(返却済み)の2状態
- `reporterName` と `returnedTo` は **一般向けAPIのレスポンスから除外**(`publicItem`関数で整形)

### 探し物リクエスト (lostRequest)

```json
{
  "id": 1,
  "keywords": "青い水筒 体育館",
  "category": "bottle",
  "color": "blue",
  "contactName": "2-3 佐藤",
  "status": "waiting",
  "matchedItemIds": [1],
  "createdAt": "2026-06-11T01:00:00.000Z"
}
```

- `status` は `waiting`(待機中)→ `resolved`(解決済み)
- 登録から **30日経過で自動的にマッチング対象外**

## 7. セキュリティ仕様

| 対策 | 実装 | 防げる攻撃 |
|---|---|---|
| 入力値検証 | 文字数制限・カテゴリ/色のホワイトリスト検証 | 不正データの混入 |
| XSS対策 | フロントで全表示値をエスケープ(`esc`関数) | スクリプト埋め込み |
| レート制限 | IPごとに回数制限(API別に10〜30回/分) | スパム登録・PIN総当たり |
| ファイル検証 | MIMEタイプ(JPEG/PNG/WebP)+8MB上限 | 巨大/不正ファイル |
| PIN認証 | 返却操作と履歴閲覧に `STAFF_PIN` | なりすまし返却・個人情報閲覧 |
| 情報の最小公開 | 名前系フィールドを一般APIから除外 | プライバシー漏えい |
| セキュリティヘッダ | `X-Frame-Options: DENY` など | クリックジャッキング等 |
| エラーの隠蔽 | スタックトレースを返さず定型JSONのみ | 内部構造の漏えい |

## 8. 環境変数(設定)

| 変数 | 必須 | 説明 |
|---|---|---|
| `PORT` | - | 待ち受けポート(既定: 3100) |
| `STAFF_PIN` | 推奨 | 保管担当のPIN。未設定だとデモモード(誰でも返却・履歴閲覧可) |
| `FIREBASE_DATABASE_URL` | - | FirebaseのDB URL。設定するとFirebase保存になる |
| `FIREBASE_KEY_PATH` | - | サービスアカウント鍵ファイルのパス(ローカル開発用) |
| `FIREBASE_SERVICE_ACCOUNT` | - | 鍵JSONそのもの(Render等のホスティング用) |
| `FIREBASE_ROOT` | - | DB内のルートノード名(既定: mitsukaru) |
| `DATA_DIR` | - | ローカル保存時のデータ置き場 |

**保存先の自動判定**: Firebase設定があればFirebase、なければローカルJSONファイル。
コードの変更なしで切り替わります。
