# 🔍 落とし物マッチングAI「みつかる君」

学校の「落とし物が持ち主に届かない」問題を解決するシステムです。
拾った人が**写真を撮るだけ**でAIが自動でタグ付けし、落とした人は**ことばで検索するだけ**。
さらに先に「探し物」を登録しておけば、合う落とし物が届いた**瞬間**にリアルタイム通知が届きます。

## 解決したい課題

- 学校の落とし物箱は中身が見えず、探しに行くのも面倒 → **大量の落とし物が放置される**
- 拾った人も「届けるのが面倒」で放置しがち → **善意が行動につながらない**
- 結果、本当は持ち主のすぐ近くにあるのに、出会えない

## 「みつかる君」の3つの体験

| 立場 | 体験 |
| --- | --- |
| 🧺 拾った人 | スマホで写真を撮るだけ。AIが「傘・黒」のように自動タグ付けして約10秒で登録完了 |
| 🔍 落とした人 | 「青い水筒」と入力するだけで、写真付きの候補が一致度順に表示される |
| 📢 まだ見つからない人 | 探し物を登録しておくと、合う落とし物が届いた瞬間に画面へ通知(音+ブラウザ通知) |

## 技術的な工夫(アピールポイント)

### 1. エッジAI(ブラウザ内で完結する画像認識)

画像認識は **TensorFlow.js + COCO-SSD** をブラウザ内で実行します。

- サーバーに画像処理の負荷がかからない(何人同時に使っても軽い)
- 外部AI APIを使わないので**コストゼロ・画像が外部に送信されない**
- 検出した物体のボックス内だけを **Canvasピクセル解析(HSV変換)** して主要色を12色に分類

### 2. 日本語シノニム辞書 + あいまい検索

「水筒/ボトル/すいとう/ペットボトル」「黒/くろ/ブラック/黒色」のような言い換え・表記ゆれを
辞書で同一視し、自由文からカテゴリと色を自動抽出します(入力中にリアルタイム判定)。

### 3. 加点方式マッチングエンジン

| 条件 | 加点 |
| --- | --- |
| カテゴリ一致 | +50 |
| 色一致 | +25 |
| キーワード部分一致 | +10×件数(最大30) |

合計が **50点以上** でマッチと判定し、Socket.IO で待機中の落とし主へ即時通知します。

### 4. 返却率の見える化

ダッシュボードで「届いた数 / 返却数 / 返却率 / カテゴリ別状況」を可視化。
社会的インパクトを**数字で**示せます。

## システム構成

```
[ブラウザ(エッジAI)]
  ├ TensorFlow.js COCO-SSD … 物体認識
  ├ Canvas HSV解析 … 主要色の抽出
  └ (HTTP / WebSocket)
        ↓
[Node.js サーバー (Express + Socket.IO)]
  ├ マッチングエンジン(シノニム辞書 + 加点方式)
  ├ 画像保存 (uploads/)
  └ JSONデータストア (data.json)
```

- **Backend**: Node.js / Express / Socket.IO / multer
- **Frontend**: Vanilla JS / TensorFlow.js(CDN)
- **DB**: JSONファイル(セットアップ不要。ネイティブ依存なしでどこでも動く)

## セットアップ

```bash
cd mitsukaru
npm install
npm start
```

→ http://localhost:3100 を開く

※ AIの物体認識モデルは初回にCDNから読み込むため、インターネット接続が必要です。
オフライン環境でも手動カテゴリ選択で全機能が使えます(AI部分のみフォールバック)。

## デプロイ(Render)

このフォルダには `render.yaml`(Blueprint)を同梱しています。

1. GitHubに `mitsukaru` リポジトリを作成してプッシュする
2. [Render](https://dashboard.render.com/) にGitHubアカウントでログイン
3. 「New +」→「Blueprint」→ `mitsukaru` リポジトリを選択
4. サービス `mitsukaru` が自動作成される(freeプラン)
5. 必要なら環境変数 `STAFF_PIN` をダッシュボードで設定(返却記録の保護)

### 本番運用の注意

- **freeプランはディスクが一時的**です。再デプロイ・再起動で登録データと画像が消えます(文化祭などの短期デモなら問題なし)
- データを残したい場合は有料プランで永続ディスクをマウントし、環境変数 `DATA_DIR` にマウント先(例: `/var/data`)を設定してください(`render.yaml` 内のコメント参照)
- HTTPSはRenderが自動で終端します

## デモのシナリオ例(発表用)

1. 審査員のペットボトルを借りて `found.html` で撮影 → AIが「ボトル・色」を自動判定
2. 別の端末(スマホ)で `lost.html` に「青い水筒」と登録 → 待機画面に
3. PCで登録を完了した瞬間、スマホに「🎉 みつかったかも!」が**リアルタイムで届く**
4. ダッシュボードで返却率が更新されるのを見せる

## セキュリティ・プライバシーへの配慮

- 写真は落とし物そのものだけを撮影する運用(登録画面でも注意喚起を表示)
- 画像認識はブラウザ内で完結し、外部のAIサービスには一切送信しない
- 名前の入力はすべて任意。登録者・受取人の名前は**APIレスポンスに含めない**(他の利用者に公開されない)
- サーバー側で入力値を再検証(カテゴリ/色のホワイトリスト・文字数制限・画像はJPEG/PNG/WebPのみ・8MBまで)
- 表示時はすべてHTMLエスケープ(XSS対策)
- レート制限(登録系は1分あたり15回まで)でスパム・連打を防止
- セキュリティヘッダ(`X-Content-Type-Options` / `X-Frame-Options` / `Referrer-Policy`)
- 探し物リクエストは30日で自動的にマッチング対象外(放置データの抑制)

### 返却記録の保護(STAFF_PIN)

そのままでは誰でも返却記録ができます(文化祭デモ向け)。
本運用では環境変数 `STAFF_PIN` を設定すると、返却記録の操作時にPIN入力が必要になります。

```bash
# Windows (PowerShell)
$env:STAFF_PIN = "1234"; npm start

# macOS / Linux
STAFF_PIN=1234 npm start
```
